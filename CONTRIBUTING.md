## 贡献流程

### 1. 报告 Bug 或功能建议

- 请在本项目的 GitHub Issues 中搜索是否已存在类似问题。
- 如不存在，请创建新的 Issue，使用模板填写详细描述：
  - **Bug 报告**：描述问题现象、复现步骤、预期行为、实际行为、环境信息（浏览器版本等）
  - **功能建议**：描述你希望增加的功能、使用场景和预期收益

### 2. 提交代码（Pull Request）

1. **Fork 本仓库** 到你的 GitHub 账号
2. **创建分支**：`git checkout -b feature/your-feature-name`
3. **编写代码**：遵循现有代码风格，保持代码简洁可读
4. **测试**：确保修改不影响现有功能，在本地浏览器中验证
5. **提交 PR**：清晰描述你的修改内容、动机和测试情况

## 项目结构

```
/
├── index.html              # 完整应用（标题从 meta 动态读取）
├── data.js                 # 题库数据
├── decrypt.js              # 配套解密脚本
├── LICENSE                 # MIT 许可证
├── README.md               # 项目说明
└── CONTRIBUTING.md         # 贡献指南
```

- `index.html` 包含所有 UI、样式和逻辑，标题和副标题从 `window.meta` 动态读取，请勿硬编码
- `data.js` 提供 `window.allQuestions` 和 `window.meta` 数据接口
- `decrypt.js` 负责数据解密（如需），具体实现请参考代码注释

## 提交规范

建议使用如下格式的提交信息：

```
<type>(<scope>): <subject>

<body>
```

**Type 类型**：
- `feat`：新功能
- `fix`：Bug 修复
- `docs`：文档更新
- `style`：代码格式调整（不影响功能）
- `refactor`：代码重构
- `perf`：性能优化
- `test`：测试相关
- `chore`：构建工具或辅助工具变动

**示例**：
```
feat(ui): 新增题型筛选功能

在抽屉列表中添加按题型筛选的选项，方便用户快速定位特定题型
```
